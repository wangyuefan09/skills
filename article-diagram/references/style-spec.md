# Draw.io 视觉规范

本文件定义 article-diagram 的统一视觉风格：Editorial Clearline。它主要面向公众号、博客和技术教程正文，目标是在电脑和手机中都保持清晰、克制和易读。

## 视觉方向

整体效果接近编辑整理过的技术文章插图：

- 白色画布、浅色填充、清晰描边和高对比文字
- 扁平、克制，默认不使用阴影、渐变和 3D 效果
- 优先使用纵向或少列布局，避免手机端把节点压得过小
- 通过语义颜色区分角色，但一张图只保留少量强调色
- 图中默认不添加作者名、个人标识、水印或装饰性签名

信息结构、移动端可读性、视觉统一的优先级依次降低。不能为了容纳更多节点而缩小字号。

## 画布与信息密度

| 场景 | 推荐画布 | 用途 |
| --- | --- | --- |
| 正文横图 | `1000 x 700` | 架构总览、横向调用关系、对比图 |
| 正文竖图 | `760 x 1100` | 流程图、状态图、移动端优先的结构图 |
| 长流程图 | `760 x 1400` | 无法拆分的长步骤流程；超过此高度优先拆图 |

- 画布外边距至少 `56px`
- 每行主节点不超过 `3-4` 个
- 单张图建议不超过 `12` 个主节点
- 超过 `16` 个主节点时，优先拆成多张图或多个 diagram page
- 公众号正文优先选择 `760px` 宽的纵向布局；只有结构天然横向时才使用横图
- 文章标题已经说明图意时，可以省略图内主标题，把空间留给结构

## 字体与文字

| 属性 | 值 | 说明 |
| --- | --- | --- |
| **Font Family** | `PingFang SC, Microsoft YaHei, Noto Sans CJK SC, Arial, sans-serif` | 优先使用常见中文字体 |
| **Font Size (Title)** | `30` | 图表主标题 |
| **Font Size (Container)** | `20` | 容器和分组标题 |
| **Font Size (Node)** | `20` | 核心节点文字 |
| **Font Size (Edge Label)** | `16` | 连线标签 |
| **Font Size (Note)** | `16` | 旁注和补充说明 |

- 节点标签尽量控制在 `1-2` 行，每行约 `4-10` 个中文字符
- 连线标签只写协议、动作或结果，通常不超过 `6` 个中文字符
- 不在节点里塞完整段落；长说明放到正文或独立旁注中
- 中文和英文混排时保留自然空格，不使用全大写长标签

## 全局常量

| 属性 | 值 | 说明 |
| --- | --- | --- |
| **Background** | `#FFFFFF` | 适配公众号正文的白色背景 |
| **Shape** | `rounded=1;arcSize=8` | 小圆角矩形 |
| **Node Stroke** | `strokeWidth=2` | 缩小后仍能看清的节点边框 |
| **Edge Style** | `edgeStyle=orthogonalEdgeStyle;rounded=1` | 圆角正交连线 |
| **Edge Width** | `strokeWidth=2` | 默认连线粗细 |
| **Edge Color** | `#64748B` | 默认箭头色 |
| **Label BG** | `labelBackgroundColor=#FFFFFF` | 连线标签保持干净 |

## 尺寸与留白

- 常规服务节点建议 `190 x 76`
- 关键入口或网关建议 `210 x 84`
- 小型工具节点建议 `160 x 68`
- 数据存储节点建议 `180 x 84`
- 节点最小横向间距 `56px`
- 节点最小纵向间距 `44px`
- 容器横向内边距约 `32px`，纵向内边距约 `44px`

| 关系 | 线型 | 颜色 |
| --- | --- | --- |
| 同步请求 | 实线，`2px` | `#64748B` |
| 异步事件 / 消息 | 虚线，`2px` | `#D97706` |
| 数据读写 | 实线，`2px` | `#7C3AED` |
| 控制 / 配置 | 虚线，`2px` | `#2563EB` |
| 错误 / 降级 | 虚线，`2px` | `#DC2626` |

## 高级设置

| 设置 | 值 | 说明 |
| --- | --- | --- |
| **Shadow** | `shadow=0` | 关闭阴影，避免缩小后边缘发脏 |
| **Gradient** | 禁用 | 保持稳定的文章视觉 |
| **Sketch Mode** | 禁用 | 除非用户明确要求草图风格 |

## 使用建议

- 一张图使用中性色加 `2-3` 种语义色，避免彩色拼盘。
- 同一语义节点保持相同填充色和边框色。
- 不只依赖颜色表达关系，同时使用文字、线型或箭头方向。
- 容器视觉要轻，核心节点视觉要明确。
- 优先绕开连线交叉；无法避免时，让次要连线从分组外围绕行。
- 导出后以正文实际宽度或约 `375px` 宽度预览，发现文字偏小就拆图，不继续缩字号。
