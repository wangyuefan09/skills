# Draw.io 视觉规范

本文件定义 wanyfon-diagram 的统一视觉风格：Wanyfon Clearline。只有在需要生成或调整图表样式时再读取。

## 视觉方向

整体效果接近整理干净的工程设计稿：

- 浅色画布、细描边、低饱和填充
- 扁平、克制，默认不使用阴影
- 通过语义颜色区分角色，不随机上色
- 中文和英文标签都保持清晰
- 排版紧凑但不拥挤，留出足够扫描空间

避免亮色渐变、3D 效果、厚重阴影、装饰性图标和过大的卡片式容器。

## 尺寸与留白

- 画布外边距至少 `48px`
- 容器横向内边距约 `28px`，纵向内边距约 `36px`
- 节点最小横向间距 `48px`
- 节点最小纵向间距 `36px`
- 常规服务节点建议 `168 x 64`
- 关键入口或网关建议 `188 x 72`
- 小型工具节点建议 `140 x 56`
- 数据存储节点建议 `150 x 64`

## 高级设置

| 设置 | 值 | 说明 |
|-----|-----|------|
| **Shadow** | `shadow=0` | 默认关闭阴影 |
| **Gradient** | 禁用 | 保持扁平和稳定 |
| **Sketch Mode** | 禁用 | 除非用户明确要求草图风格 |

## 使用建议

- 同一张图里，相同语义的节点颜色保持一致。
- 一张图通常使用中性色加 2-3 种语义色，避免变成单一色调或彩色拼盘。
- 标题字号只用于图表主标题；容器、节点和连线标签使用各自常规字号。
- 容器视觉要轻，核心节点视觉要明确。
- 强调重要节点时只增加边框粗细，不使用发光或大阴影。
- 优先绕开连线交叉；无法避免时，让次要连线从分组外围绕行。
